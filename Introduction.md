# 项目简介：PowerPoint 风格网页一致性生成系统

本项目的核心使命是开发一个基于大语言模型（LLM）的智能网页生成系统，将传统的 PowerPoint 演示文稿制作体验转化为高性能、跨平台的网页格式（HTML/Markdown）。系统通过“模板约束 + LLM 生成”的混合方案，解决传统 PPT 制作耗时、多页风格难统一以及跨平台分发受限的痛点。

## 1. 项目功能模块详细描述

根据项目章程与需求分解，系统分为以下四个核心功能模块：

- 结构化表示框架 (Structured Representation Framework)： 这是系统的基础底座，包含布局原子库（捕获基本布局模式）、组件规范（标准化内容元素表示）以及样式令牌（Style Tokens）。它负责将复杂的视觉设计属性（如色彩、字体、间距）编码为可复用的数据结构，确保生成内容的逻辑一致性。

- 自动化布局选择与推理引擎 (Reasoning Engine)： 该模块充当“逻辑大脑”，根据输入内容的语义信息（如关键点数量、是否含有数据图表等），采用约束求解、启发式搜索或 LLM 推理机制，为每一页自动匹配最合理的布局并智能放置组件。

- 基于 LLM 的代码生成模块 (Code Generation Module)： 利用 LLM 的代码生成能力，结合系统提示词（Prompt）工程与样式令牌注入，将语义内容转化为符合预设设计规范的网页代码。该模块负责最终输出 HTML 或 Markdown 格式的网页，确保生成结果在保持美观的同时，实现多页间的全局视觉一致性。

- 量化评估框架 (Quantitative Evaluation Framework)： 这是一个自动化质量控制系统，用于评估生成布局的合理性（如检测元素重叠率为 0%）和视觉风格的偏差（如全局色差控制在 5% 以内）。评估结果不仅用于最终验证，还为系统的持续迭代提供反馈信号。

## 2. 项目整体框架搭建与技术栈选用

**整体架构设计 (Pipeline)**

系统采用模块化的端到端流程：内容输入与语义提取 → 布局推理与组件匹配 → 代码生成与样式注入 → 量化评估与反馈。

**技术栈选用** 

大语言模型能力：核心驱动力为 LLM API（如 DeepSeek），用于处理自然语言指令、进行语义布局推理及生成代码。
后端/算法层：
开发语言：使用 Python 或 Node.js（基于 LLM 生态的普遍选择）。
功能：实现布局原子库管理、约束求解算法以及 Prompt 调度逻辑。
前端展示层：
技术：HTML5, CSS3, JavaScript。
功能：构建 html/markdown 模板库，开发用户交互原型界面，实现网页预览功能。
工程化工具：采用 Style Token 系统（如 CSS Variables 或 JSON Tokens）来控制关键设计属性，确保跨页面视觉高度统一。

**请按照以下目录结构初始化项目：**

/framework: 定义样式令牌（tokens）、布局原子（layouts）和组件规范。
/engine: 实现基于语义的布局匹配逻辑与约束求解。
/generator: 编写 LLM Prompt 策略及 HTML/Markdown 代码生成器。
/evaluator: 编写布局合理性（重叠检测）与一致性检测脚本。
/demo: 搭建基于网页的前端预览与内容上传界面。
